## 📧 이메일/슬랙용 (자세한 버전)

```
안녕!

AI 토큰 절약 도구 'Token Optimizer' 프로토타입 만들었어. 한번 테스트해줘.

🌐 사이트: https://dandanyoou.github.io/dancaveman/

📌 두 가지 도구:

1. 🦣 Caveman — 프롬프트 압축
   • 긴 한국어 프롬프트를 "원시인 말투"로 압축
   • 평균 70% 토큰 절감
   • 사용처: ChatGPT, Claude 등에 길게 쓰는 사람

2. 🍩 Homer — CLI 출력 압축  
   • git log, npm test, JSON 등을 LLM-ready로 변환
   • 평균 81% 절감 (rtk 능가)
   • 사용처: AI 코딩 도구 (Claude Code, Cursor 등)

🧪 테스트해주면 좋은 것:

A) Caveman
- 평소 ChatGPT에 길게 쓰는 메시지 한두개 압축
- 압축본을 그대로 ChatGPT에 보내서 같은 답변 나오는지 비교
- "이건 너무 압축됐다" / "이건 적당하다" 표시

B) Homer
- 터미널에서 명령어 실행 → 결과 복사 → Homer 탭에 붙여넣기
- 추천 명령: `git log -10`, `npm test`, `curl -s api.github.com/users/torvalds`
- 압축본을 Claude Code에 넣어보고 정상 답변 나오는지

✏️ 후기 양식:
- 실제 토큰 절약: __% (사이트가 표시함)
- AI 답변 품질 변화: 좋음 / 비슷 / 나쁨
- 가장 좋은 점: ___
- 가장 어색한 점: ___
- 더 필요한 기능: ___

이번주 안에 답변 줄 수 있으면 진짜 고마워! 🙏
```

## 🎯 피드백 수집 폼 (Google Form 추천)

직접 만든다면 다음 항목 추천:

1. **어떤 도구를 사용했나요?** (체크박스)
   - [ ] 🦣 Caveman (프롬프트)
   - [ ] 🍩 Homer (CLI 출력)
   - [ ] 둘 다

2. **압축률 만족도** (1-5)
3. **AI 답변 품질** (압축 전 vs 후)
   - 더 좋아짐
   - 비슷함
   - 나빠짐
   - 비교 안 함

4. **실제로 계속 사용할 의향** (1-10)

5. **자유 의견** (텍스트)
   - 어떤 케이스에서 잘 안 됐나요?
   - 추가하면 좋을 기능?
   - 어디에 공유하면 좋을까요?

6. **본인 정보** (선택)
   - 주 사용 AI: ChatGPT / Claude / Gemini / 기타
   - 사용 빈도: 매일 / 주 3-4회 / 주 1-2회 / 가끔
